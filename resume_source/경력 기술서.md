### **마이보이스 (User App + 덥라이트 제작 파이프라인) 더빙 체험 기능 개발**

2026.05 ~ 2026.06

**스트리밍형 더빙 PoC + 웹/모바일 녹음 플로우 + 독립 플레이어 개편 + 덥라이트(검수) 기반 콘텐츠 제작·연동 파이프라인 구축**

🏆 **주요 성과**

- **“제작(덥라이트) → 노출(관리자) → 이용(이용자 앱)” 전체 파이프라인 구축**: 검수 페이지에서 녹음 구간·크롭 이미지·말풍선 좌표·주요 캐릭터를 생성하면 content-hub 저장 → 관리자 작품 생성 → 이용자 대사 선택/녹음으로 이어지는 단일 흐름 설계·구현
- **연출/대사 타임라인 어긋남을 “마커(시간)+픽셀(이미지)” 이중 기준으로 해소**: spoint가 절대 기준이 되기 어려운 한계를, 시간 기준 대사 수집과 픽셀 기준 이미지 크롭으로 분리해 정합성 확보
- **추정 매핑을 직접 좌표 태깅 워크플로우로 전환**: 줌 뷰어에서 대사 순서대로 말풍선을 페어링하고, 절대 px + 비율(%) 이중 저장으로 관리자↔이용자 렌더 스케일 차이에서도 위치 안정화
- **스트리밍 더빙 체험 기술 리스크를 PoC 기준으로 선제 정의** 및 **웹/모바일 녹음 플로우·독립 플레이어 구조 개편**으로 이후 확장 시 결합도 감소

💼 **주요 업무**

- **[이용자 앱]** 스트리밍 더빙 기술 검토·PoC(동기화·네트워크·저장·보안 URL·progressive MP4), 웹/모바일 녹음 플로우 구현, 플레이어 API 연동, 독립 플레이어/탭 구조 정리
- **[덥라이트 제작]**
    - **마이보이스 설정 CRUD + 구간 정의 로직**: 생성/수정/삭제/조회·플레이어 연동, 마커 기반 구간 시간 계산·겹침 검사·대사 홀 수집, spoint 엣지케이스(관통 대사 제외·타이 보강) 개선
    - **구간 추출/크롭/태깅 UI**: 모달→seamless 사이드바 전환·칩형 요약·raw 세트 데이터, 크롭 확대 뷰어, 좌우 분할 줌 뷰어 좌표 태깅(% → px → 이미지 분기)
    - **content-hub API 연동**: 씬·좌표 생성/조회/삭제, presigned URL S3 업로드, buildOnly/onApply 분리로 중복 생성 방지
    - **픽셀 크롭 및 비율 좌표 저장**: 파란 줄 기반 startMpoint/endMpoint·다중 이미지 누적 크롭, 절대 px + 비율(%) + 씬 크기 저장 및 하위 호환
    - **주요 캐릭터 지정 + QA**: 캐릭터 목록·썸네일·is-main 저장/잠금/복원, 구간·크롭·태깅·캐릭터 전반 검증

### **보고팡 (React + Next.js) B2G 실서비스 개발**

2026.02 ~ 2026.04

**B2G 이용자 서비스 실서비스 구현 + 시범/운영 듀얼 모드 아키텍처 + 플레이어·녹음 안정화 + 자체 애널리틱스(Umami) 구축**

- **B2G 핵심 정책/기술 리스크를 "구현 기준"으로 선제 확정**: SSO·ILS 연동, 공공 접근성/보안, 통계·정산까지 기술검토 항목으로 구조화해 개발 중 정책 번복 리스크를 줄임
- **대출·예약 시스템을 '운영 가능한 방식'으로 설계 확정 후 UI/동작까지 일관 구현**: 번호표 기반 예약 + 반납 시 자동 대출, 실시간 대출/예약 상태 노출·한도/예외 케이스까지 UX 확정 문서 기준으로 구현 범위를 고정
- **시범(Trial) ↔ 운영(Operational) 듀얼 모드 아키텍처 도입**: 서브도메인 기반 모드 판별 + 공개 조회 API + AppAuthGate 무로그인 통과 + 라우트 이동 시 재조회를 결합한 권장 아키텍처를 설계·구현. 환경변수 플래그 기반 안전 롤아웃과 새로고침 시 모드 확정 전 렌더 차단(flicker 제거)으로 시범 UX 신뢰도 확보
- **체험하기 수동 매칭 시스템 도입으로 썸네일/마커 정합성 이슈 해소**: 추정 기반 매핑 한계를 이미지-대사 직접 페어링 워크플로우(HoleDrawNew)로 전환, 관리자 → 이용자 앱 → 백오피스 미리보기 플레이어 3 surface까지 직접 매핑 우선 + 추정 fallback의 이중 안전 구조로 동기화
- **플레이어/녹음 기능을 실사용 가능한 '프로세스'로 완성하고 회귀 이슈 일괄 해소**: 플레이어 이식·라우팅·콘텐츠 로딩 연결 + 녹음 모달/장면·마커 컷 싱크 재작성/모바일 몰입·뒤로가기 재생(레이스 방지)/로딩 실패 자동 재시도+복구 UX/미대출 상태 스낵바 정책까지 안정화
- **Umami 기반 자체 애널리틱스를 '운영 가능한 분석 체계'로 구축**: 프론트 이벤트 트래킹 유틸/패턴 정리, EC2 + Docker 기반 Umami 서버 구축/복구 가이드 문서화. B2B 영업 지표 4종(view_sales_page 등) 자동 계측 + 운영팀 셀프 리포팅 가이드까지 패키지로 제공

💼 **주요 업무**

- **B2G 인증/연동 기술검토 및 핵심 플로우 구현**: SSO·보안·운영 제약 체크리스트 정리, 대출·예약·내서재·검색 등 이용자 핵심 플로우 구현 기준 확정 및 반영

- **플레이어 통합·녹음 기능 구현 및 안정화 스프린트**: 모바일/태블릿/데스크톱 시안 기반 UI/UX 재정비, 녹음 패널/카드 UI + 저장/적용 분리 로직, 미대출 상태 스낵바 정책, 컷 싱크/모바일 탭/뒤로가기 재생/로딩 복구 등 회귀 이슈 안정화
- **자체 애널리틱스(Umami) 구축 및 영업 지표 패키징**: 연동 환경변수/스크립트 주입/이벤트 유틸 정리, 서버 구축(EC2/Docker), B2B 영업 지표(view_sales_page/view_works_list/view_experience_page/view_preview_page) 표준화 및 운영팀 가이드 문서화
